# 📑 시니어 엔지니어 기술 평가서 (Technical Review)

**작성자:** Senior Software Engineer (GitHub Copilot)  
**대상 프로젝트:** Image Deco Studio  
**평가 일자:** 2024년 12월 20일

## 1.1 총평 (Executive Summary)

> **"훌륭한 프로토타입이나, 프로덕션 레벨로 가기 위한 리팩토링이 시급함"**

본 프로젝트는 외부 라이브러리 의존도를 최소화하고 Vanilla JS(순수 자바스크립트)로 핵심 기능(드래그 앤 드롭, 트랜스폼)을 직접 구현했다는 점에서 높은 점수를 줄 수 있습니다. 기획 의도인 "가벼운 웹 편집기"에 부합하는 기술 스택 선정입니다.

하지만, **전역 변수 기반의 상태 관리**와 **관심사 분리(Separation of Concerns)의 부재**는 향후 기능 확장 시 치명적인 기술 부채(Technical Debt)가 될 가능성이 높습니다.

## 1.2 상세 기술 평가

### 🏗 아키텍처 및 구조 (Score: B-)
*   **장점:**
    *   HTML/CSS/JS가 명확히 분리되어 있고, 페이지별(`editor`, `result`, `start`)로 스크립트가 나뉘어 있어 파일 구조가 직관적입니다.
    *   `js/editor.js`에서 외부 라이브러리 없이 DOM 조작만으로 에디터를 구현한 것은 브라우저 동작 원리에 대한 이해도가 높음을 시사합니다.
*   **단점:**
    *   **모듈 시스템 부재:** ES6 Modules(`import`/`export`)를 사용하지 않고 `<script src="...">` 방식에 의존하고 있습니다. 이는 네임스페이스 오염을 유발합니다.
    *   **컴포넌트화 부재:** 스티커 객체, 캔버스 영역, 저장 로직 등이 하나의 파일에 혼재되어 있습니다.

### 💻 코드 품질 (Score: C+)
*   **Global State 오남용:**
    *   `js/editor.js`의 `selectedSticker`, `action`, `startX` 등이 전역 변수로 선언되어 있습니다. 앱이 복잡해지면 상태 추적이 불가능해집니다.
*   **이벤트 핸들링:**
    *   `window.addEventListener("mousemove", ...)` 방식은 드래그 끊김을 방지하는 좋은 패턴이나, 이벤트 리스너를 제거(remove)하는 로직이 명확하지 않아 메모리 누수 가능성이 있습니다.
*   **하드코딩:**
    *   스티커 경로 배열(`stickers`)이 코드 내부에 하드코딩되어 있어, 추후 서버 연동 시 수정이 필요합니다.

### 🎨 UI/UX 구현 (Score: A)
*   **완성도:**
    *   `css/editor.css`의 Glassmorphism(유리 효과) 디자인과 `js/start.js`의 캔버스 별 애니메이션은 사용자 경험을 크게 향상시킵니다.
    *   반응형 미디어 쿼리(`@media`)를 적절히 사용하여 모바일 환경을 고려한 점이 우수합니다.
